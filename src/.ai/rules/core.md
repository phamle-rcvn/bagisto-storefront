# Core Rules

- Prefer the smallest correct change.
- Inspect existing source and tests before implementation.
- Preserve public behavior unless the task explicitly changes it.
- Do not invent requirements, domain models, API fields, or hidden business rules.
- Use English for code, identifiers, filenames, tests, and code comments.
- Keep comments short and explain non-obvious intent.
- Do not add dependencies without explicit approval.
- Do not edit generated output or dependency directories.
- Never commit secrets, credentials, tokens, or local environment files.
- Follow `.ai/rules/coding-standards.md` for naming and comments.
- Follow `.ai/rules/git.md` for the absolute prohibition on Git mutations.
