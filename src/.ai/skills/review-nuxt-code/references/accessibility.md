# Accessibility Review

- Prefer native semantic elements before custom interactive containers.
- Verify interactive controls have accessible names and correct roles.
- Check keyboard operation, focus visibility, focus order, and focus restoration.
- Verify form controls have labels, instructions, and associated error messages.
- Check headings and landmarks describe page structure without skipped semantic intent.
- Verify images have appropriate alternative text or are correctly marked decorative.
- Check status, validation, loading, and error changes are perceivable without relying only on color.
- Verify disabled behavior does not trap focus or block necessary explanation.
- Treat automated checks as supporting evidence, not a replacement for interaction review.
